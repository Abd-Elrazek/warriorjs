Publier

Ceci est la structure de base d'un package de tour :

warriorjs-tower-got
├── index.js
└── package.json

index.js contient le code utilisé tout au long de ce guide. package.json contient quant à lui les informations de package npm :

{
  "name": "warriorjs-tower-got",
  "version": "0.1.0",
  "description":
    "There is only one war that matters: the Great War. And it is here.",
  "main": "index.js",
  "keywords": ["warriorjs-tower"],
  "dependencies": {
    "@warriorjs/geography": "^0.4.0"
  }
}

Attention :

  • Le nom du package doit commencer par warriorjs-tower- pour que la tour soit automatiquement chargée par WarriorJS.
  • warriorjs-tower doit figurer dans le champ "mots-clés" pour rendre votre tour plus facile à localiser.

En travaillant sur une tour, vous pouvez utiliser le pack npm pour créer un tarball associé, puis l'installer au même endroit que @warriorjs/cli de cette façon :

npm install <path/to/tarball>

Votre tour devrait être chargée automatiquement en exécutant warriorjs.

Une fois que vous avez testé et ajusté votre tour, vous pouvez la publier sur npm pour que d'autres joueurs en profitent. Suivez ce guide pour apprendre comment à publier un package sur npm.
